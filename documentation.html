Dokumentation

Erklärung der wichtigsten Quelltext-Passagen.

Diese Seite zeigt, wo die geforderten Punkte im Projekt umgesetzt wurden.

1. Mindestens drei Fetch-Abfragen

In js/app.js gibt es mehrere Funktionen mit fetch(): fetchEmission() fragt die Climatiq API für CO2-Werte ab. fetchSupportLevels() lädt eine Liste von Supporter-Leveln. fetchPremiumMessage() lädt eine Bestätigungsnachricht nach der Spende.

2. Liste ausgeben und Array durchlaufen

Die Funktion renderSupportLevels() filtert ein Array und durchläuft es mit forEach(). Für jedes Element wird eine Karte im HTML-Bereich supportLevels erzeugt.

3. Filter mit Formularfeldern

Auf support.html gibt es ein Suchfeld levelSearch. Bei jeder Eingabe wird die Supporter-Liste neu gefiltert, damit nur passende Level angezeigt werden.

4. Local-Storage

Nach dem Spendenformular speichert saveSupporter() Name, E-Mail, Betrag, Monate und den Premium-Status in localStorage. Der Rechner liest diesen Wert und schaltet Zug und Flug frei.

5. Premium-Funktionen

In calculatorForm wird vor der Berechnung geprüft, ob das gewählte Verkehrsmittel Premium braucht. Ohne gespeicherte Unterstützung zeigt die Seite einen Hinweis und verweist auf das Formular.
